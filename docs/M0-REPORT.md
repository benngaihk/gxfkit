# M0 Spike — Go / No-Go Report

**Date:** 2026-06-30
**Subcommand:** `gff2gtf` (vs `agat_convert_sp_gff2gtf.pl`)
**AGAT baseline:** 1.7.0 (`quay.io/biocontainers/agat:1.7.0--pl5321hdfd78af_0`)
**Decision: GO.** The gate (≥10× wall-clock on ≥1 file **and** a clear path to
output parity) is met with large margin.

---

## The gate

> M0 stops the project if the first subcommand can't reach ≥10× wall-clock
> speedup, or if there's no clear path to output parity.

Both conditions are met:

- **Speed:** 63–89× faster wall-clock across the corpus; memory 3.9–4.9× lower.
  See below.
- **Parity path:** gxfkit already reproduces AGAT's core GFF→GTF transform. The
  residual differences are few, **systematic, and fully enumerated** in
  [PARITY.md](PARITY.md) — each is a deterministic AGAT rule, not an open
  research question.

## Headline measurement (reproducible harness, `benchmark/run.sh`)

Both tools run in the **same Linux container**, timed with `/usr/bin/time`
(best of 2 cold runs — AGAT won't overwrite an output file, so each run deletes
it first). Pinned Ensembl release 110 input.

| file (lines)            | AGAT 1.7.0 | gxfkit 0.0.1 | speedup   | AGAT mem | gxfkit mem |
|-------------------------|------------|--------------|-----------|----------|------------|
| yeast (35.9k)           | 9.68 s     | 0.12 s       | **80.7×** | 752 MB   | 155 MB     |
| human chr21 (40.8k)     | 12.52 s    | 0.14 s       | **89.4×** | 935 MB   | 194 MB     |
| human chr1 (322.3k)     | 78.47 s    | 1.25 s       | **62.8×** | 5.47 GB  | 1.41 GB    |

This table is regenerated by the harness and mirrored at the top of the
[README](../README.md) / `benchmark/results/summary.tsv`. (An independent clean
`/usr/bin/time -v` run on chr1 measured 75.87 s vs 1.04 s — same order, normal
run-to-run variance.)

## Parity status (honest baseline)

Post-normalization parity on the pinned corpus is **~73% (yeast)** down to
**~31% (human)** at the end of M0. This is a *starting* baseline, not a final
number: the spike implements the conversion core, and every remaining gap is a
known AGAT behavior listed in [PARITY.md](PARITY.md) (synthesized exon IDs,
`agat-<type>-<N>` IDs for ID-less features, multi-value `tag`, etc.). Closing
them to the ≥95% target is the M1 work item and requires no new ideas.

We deliberately report the low baseline rather than cherry-pick: credibility
comes from the reproducible harness and the documented divergence ledger.

## What was built in M0

- Rust workspace: `gxfkit-core` (parser/model/convert) + `gxfkit` CLI.
- Working `gff2gtf` (hierarchy resolution, Ensembl prefix stripping, attribute
  carry-over), unit-tested.
- Pinned public corpus (`corpus/download.sh`).
- Reproducible benchmark: one Docker image with AGAT + gxfkit
  (`benchmark/run.sh`) — same container, no cross-OS confounders.
- Parity normalizer + divergence ledger.

## Risks confirmed (matches the plan)

- **Edge-case compatibility is the bulk of the work.** Confirmed: the happy path
  was quick; matching AGAT's feature-mangling quirks is where M1–M3 effort goes.
- **Memory:** gxfkit currently owns every field (reads whole file). Already ~4×
  better than AGAT on the large file, but the streaming/borrowing optimization
  (M1) should widen this further.

## Recommendation

Proceed to **M1**: close the documented parity gaps to ≥95%, add the parity
regression to CI, and do the first soft release. The economics are exactly as
the plan predicted — no algorithmic risk, just disciplined diff-driven work.
